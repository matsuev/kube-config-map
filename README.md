# kube-config-map

### Задание для самостоятельной работы:


В репозитории в папке __./bin__ лежат три исполняемых файла:
   - ./bin/appapi
   - ./bin/appinfo
   - ./bin/applogin

Эти файлы нужно упаковать в Docker контейнеры:
   - appapi:latest
   - appinfo:latest
   - applogin:latest

При запуске контейнеров в них передаются три переменные окружения:
   - APP_NAME: <Название приложения> (любое текстовое значение)
   - LOG_LEVEL: <Уровень логирования> (варианты: "none", "info", "warning", "error", "debug", "trace")
   - BIND_PORT: <Номер порта> (порт, на котором приложение будет принимать сетевые подключения)

## Сделать:

1. ConfigMap's для передачи переменных окружения в Pod (для каждого контейнера)

2. ConfigMap для контейнера Nginx. В конфигурации настроить проксирование в контейнеры приложений:
   - /api в контейнер appapi на порт :7001
   - /info в контейнер appinfo на порт :7023
   - /login в контейнер applogin на порт :7058

3. Deployment для запуска приложения с контейнерами:
   - nginx:latest
   - appapi:latest
   - appinfo:latest
   - applogin:latest
